Methodology

This project benchmarks algorithms available in the pytorch_optimizer library. The evaluation process for each optimizer consists of the following steps:

  • Hyperparameter Tuning: For every function, the Optuna framework is used to search for optimal hyperparameters (e.g., learning rate, momentum) over a fixed number of trials.
  • Execution: The optimizer is run on the 2D function using the best hyperparameters found during the tuning phase.
  • Visualization: The optimization trajectory (path taken from start to finish) is recorded and plotted on the function's contour map.
  • Scoring: Optimizers are ranked based on their final distance from the global minimum and the number of iterations required to converge.

FAQ

What functions are used?
The suite uses 18 standard optimization test functions, including multimodal functions (e.g., Ackley, Rastrigin) and valley-shaped functions (e.g., Rosenbrock). These are continuous, 2D mathematical surfaces.
Does this predict Neural Network performance?
No. These are low-dimensional (2D) problems. Deep Learning involves high-dimensional optimization with different landscape characteristics (e.g., saddle points). This benchmark strictly evaluates algorithmic behavior in a 2D context.
How are the rankings calculated?
Average Rank is the mean position of an optimizer across all 18 functions. Weighted Error Rate is the aggregated distance to the global minimum, weighted by the difficulty of the specific function.
